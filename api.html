Application Programming Interface – Jembatan antar aplikasi

Apa itu API?

API (Application Programming Interface) adalah jembatan yang menghubungkan berbagai teknologi (Database, Mobile, Game). Tanpa API, aplikasi hanyalah "pulau terpencil" yang tidak bisa berkomunikasi dengan dunia luar.

Customer (Client)

Kamu memesan makanan (Request)

Pelayan (API)

Mengambil pesanan, menyampaikan ke dapur, membawa makanan kembali

Dapur (Server/Database)

Tempat makanan disiapkan (Data)

Analogi Restoran: Client memesan (request), API sebagai pelayan mengantarkan ke server, lalu server mengembalikan response (makanan).

Jenis API yang Wajib Diketahui

REST API

Paling umum, gunakan JSON. Ringan & standar industri.

GraphQL

Client tentukan data yang dibutuhkan. Efisien.

Webhooks

API otomatis, misal bot Telegram.

Cara Kerja API (Request-Response Cycle)

4 Metode Utama (CRUD):

GET SELECT (ambil data)
POST INSERT (kirim data baru)
PUT/PATCH UPDATE (ubah data)
DELETE DELETE (hapus data)

HTTP Status Code yang umum:

200 OK: Berhasil 201 Created: Data dibuat 404 Not Found: Alamat/data tidak ada 500 Internal Server Error: Error server

Kualifikasi Backend/API Developer (Global)

  • Security (JWT/OAuth) – Mengamankan API.
  • API Documentation – Panduan jelas (Swagger/Postman).
  • Rate Limiting – Batasi request agar server tidak tumbang.
  • Versioning – Kelola perubahan API (v1, v2).
